# Concept: Streaming vs. Invocation

LangGraph provides two main ways to execute your graph: `.invoke()` and `.stream()`. Choosing the right one is critical for the User Experience (UX).

## 1. app.invoke()
Runs the graph from start to finish (or until a breakpoint) and returns the final state.

- **Best for**: Simple request-response cycles, background tasks, or when you only want to update the UI once the entire "AI turn" is complete.
- **Example**: Used in **Lesson 9** to prevent terminal flicker during tool execution.

## 2. app.stream()
Yields intermediate values as the graph executes.

- **stream_mode="values"**: Yields the full state after every node execution.
- **stream_mode="updates"**: Yields only the changes (updates) produced by the last node.
- **Token Streaming**: Yields individual words/tokens as they are generated by the LLM (requires `astream_events`).

### Why use Streaming?
1. **Perceived Latency**: The user sees the AI is working immediately rather than waiting for a large block of text.
2. **Transparency**: You can show the user exactly which node is currently running (e.g., "AI is searching Google...", "AI is calculating...").
3. **Interactive Debugging**: Essential for developers to see where a graph might be looping or stuck.

## Comparison

| Feature | .invoke() | .stream() |
|---|---|---|
| **Output** | Final result only | Flow of events/states |
| **UX** | "Thinking" indicator | Real-time updates |
| **Complexity** | Simple | Requires iteration logic |
