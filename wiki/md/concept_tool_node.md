# Concept: ToolNode

`ToolNode` is a prebuilt utility in LangGraph (`langgraph.prebuilt.ToolNode`) that simplifies the execution of tool calls generated by an LLM.

## Purpose
In a ReAct agent, when the LLM generates a tool call (e.g., `calculate(a=5, b=10)`), something needs to actually run the Python function and return the result to the graph. `ToolNode` automates this process.

## Usage
You initialize a `ToolNode` with a list of tools and add it to your graph as a standard node.

```python
from langgraph.prebuilt import ToolNode

# 1. Define tools
tools = [add, subtract, multiply]

# 2. Create the node
tool_node = ToolNode(tools)

# 3. Add to graph
workflow.add_node("action", tool_node)
```

## Key Features
1. **Multi-Tool Support**: It can handle multiple tool calls in a single turn.
2. **State Integration**: It automatically produces `ToolMessage` objects, which are added to the `messages` list in the state (if using `add_messages`).
3. **Error Handling**: It can be configured to catch exceptions and return the error message to the LLM so it can "self-correct".

## Under the Hood
When the graph reaches the `ToolNode`, it:
1. Looks at the last message in the state.
2. Identifies any `tool_calls`.
3. Executes the corresponding Python functions.
4. Appends a `ToolMessage` to the state for every result.
